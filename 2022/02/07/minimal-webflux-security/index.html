1. WebFlux Security

Spring Security对WebFlux的支持依赖于一个WebFliter，它的工作原理与Spring WebFlux和Spring WebFlux.Fn一样。你可以找到几个示例应用程序，演示了下面的代码:

  • Hello WebFlux hello-webflux

  • Hello WebFlux.Fn hello-webflux-fn

  • Hello WebFlux Method hello-webflux-method

Spring Security’s WebFlux support relies on a WebFliter and works the same for Spring WebFlux and Spring WebFlux.Fn.

  • Hello WebFlux spring-security/samples/boot/hellowebflux at 5.2.0.RELEASE · spring-projects/spring-security · GitHub

  • Hello WebFlux.Fn spring-security/samples/boot/hellowebfluxfn at 5.2.0.RELEASE · spring-projects/spring-security · GitHub

  • Hello WebFlux Method spring-security/samples/boot/hellowebflux-method at 5.2.0.RELEASE · spring-projects/spring-security · GitHub

1.1 Minimal WebFlux Security Configuration

WebFlux Security的最小配置:

You can find a minimal WebFlux Security configuration below:

这个配置提供了表单和http基本身份验证，设置授权以要求经过身份验证的用户访问任何页面，设置默认登录页面和默认注销页面，设置与安全相关的HTTP头部、CSRF保护等等。

This configuration provides form and http basic authentication, sets up authorization to require an authenticated user for accessing any page, sets up a default log in page and a default log out page, sets up security related HTTP headers, CSRF protection, and more.

注意：withDefaultPasswordEncoder() 方法不安全，官方已经废弃，只在例子中使用。
